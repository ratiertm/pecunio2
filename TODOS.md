## Phase 1

### 온보딩: 첫 로그인 시 학습 먼저 유도, 레슨 2개 완료 후 매매 화면 해금
- **Why:** 초보자에게 매매 시뮬레이터를 바로 보여주면 혼란스럽다. 기초 개념(주식이란?, 매수/매도)을 먼저 배워야 매매 화면이 의미가 있다.
- **Pros:** 사용자 이탈 감소, 편향 교육 효과 극대화, 학습→실전 자연스러운 흐름
- **Cons:** 매매 화면을 빨리 보고 싶은 사용자에게 제한적일 수 있음 (건너뛰기 옵션으로 해결)
- **Context:** Outside Voice에서 지적. 디자인 문서 Screen 1-3 순서는 유지하되, 온보딩 플로우에서 `/learn` → Chapter 1 Lesson 1부터 시작하도록 유도. Lesson 2까지 완료하면 `/trade` 해금.
- **Depends on:** 학습 시스템 구현 완료

### 편향 감지 2단계: 10건=패턴 관찰(점수 없음), 30건=수치화된 점수 활성화
- **Why:** 10건의 매매 데이터로 상관관계를 측정하면 통계적으로 noise. 30건 이상이어야 의미 있는 패턴.
- **Pros:** 잘못된 점수로 사용자를 오도하지 않음, 신뢰할 수 있는 분석 제공
- **Cons:** 초기 사용 기간에 "편향 점수"가 없어 동기부여가 떨어질 수 있음
- **Context:** Outside Voice에서 지적. 10-29건: "당신은 하락 종목을 더 빨리 팔고 있어요"(정성적 관찰). 30건+: "손실 회피 점수: 45/100"(정량적 점수). 실거래 전환 기준 100건은 유지.
- **Depends on:** 편향 분석 엔진 구현

## Phase 2

### 확증 편향: 외부 데이터 기반으로 재설계 (Finnhub 뉴스 감성 + 검색 패턴)
- **Why:** Claude가 생성한 대화를 Claude가 평가하면 순환적. 외부 정보원 기반이어야 진짜 편향을 측정할 수 있다.
- **Pros:** 실제 사용자 행동(어떤 뉴스를 읽었나, 어떤 종목을 검색했나) 기반으로 정확한 측정
- **Cons:** Finnhub 뉴스 감성 데이터에 의존, 앱 내 뉴스/검색 기능 필요
- **Context:** Outside Voice에서 핵심 통찰로 지적. Finnhub은 뉴스 감성 분석을 무료로 제공. 사용자가 특정 종목에 대해 긍정 뉴스만 읽는 패턴을 추적 가능. 앱 내 "종목 뉴스" 탭 추가 필요.
- **Depends on:** Finnhub API 연동 (US market), 앱 내 뉴스 피드 기능
